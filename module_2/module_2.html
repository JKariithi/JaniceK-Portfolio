Introduction to Embedded C

Embedded Systems

Embedded systems is a combination of hardware and software designed for a specific function. Basic hardware in embedded systems consists of: Sensors, Analog-to-digital converters, Processors, Digital-to-analog converters and acutuators. Types of embedded systems in terms of functional requuirements: Mobile, Networked, Standalone and Real-time embedded systems. Commmon software architechture in embedded systems: Simple, Interrupt, Cooporative multitasking and Preemptive multitasking/ multithreading controlled loops. Embedded systems trends: AI and ML, Edge computing, Security, Increases connectivity and Quantum Computing.

IDEs

An IDE for embedded systems combines various tools needed for software development into one application, simplifying the process.

Key components:

  • Code Editor: Provides a platform for writing and editing source code.
  • Compiler: Translates human-readable code into machine code that the embedded system can understand.
  • Debugger: Helps identify and fix errors in the code by allowing developers to step through the code, inspect variables, and set breakpoints.

In this section we looked at Arduino and Thonny IDEs.

Key Differences between Arduino and Thonny IDE

Feature Arduino IDE Thonny IDE
Programming Language C/C++ based Python (includes MicroPython support)
Target Devices Arduino boards (e.g., Uno, Nano) Python boards (e.g., Raspberry Pi Pico)
User Level Beginner to intermediate in electronics Beginner-friendly for Python learners
Built-in Debugging Limited (basic serial monitor) Yes (step-by-step visual debugger)
Main Use Case Embedded hardware projects and prototyping Learning Python and programming microcontrollers

C programming language

C is a powerful, low-level language that gives developers fine-grained control over system resources. It is most commonly used in operating systems, embedded systems, and performance-critical applications because of its speed and efficiency.
